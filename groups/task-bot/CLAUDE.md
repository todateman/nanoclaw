## タスク検出ルール

### 検出すべきパターン

1. **完了報告**: 「〜しました」「〜完了」「〜できました」「〜マージしました」「〜届きました」
   → 既存タスクの進捗追記、場合によってはステータスを「完了」に変更
2. **作業報告**: 「〜を修正した」「〜を加工した」「〜を取り付けた」「〜を設計した」「〜を出力した」
   → 既存タスクの進捗追記、またはステータスを「進行中」に変更
3. **新規作業宣言**: 「〜しようと思います」「〜する予定」「〜を作り直す」「〜やってみる」
   → 既存タスクの進捗追記、または新規タスクとして追加
4. **問題解決**: 「〜の問題を解消」「〜を修正」「〜に気づいたのでやり直した」
   → 既存タスクの進捗追記
5. **調査・検討**: 「〜を調べてみます」「〜が売ってますね」「〜を試してみよう」
   → 新規タスク（ステータス: 未着手 or 進行中）

### スキップすべきパターン

- 雑談・感想・リアクション（「いいやん！」「カッコいい」「👍」等）
- 質問への回答で作業を伴わないもの
- URLの共有のみ（ただし購入や調査の文脈がある場合は検出対象）
- bot自身の投稿（task-bot, GitHub等）

### 判断が難しい場合

「作業をした」という事実が読み取れるかどうかで判断する。
迷った場合は **検出しない** 方に倒す（過検出より見逃しの方がまし。見逃しは日次バッチで補完される）。

## 既存タスクとの照合手順

1. まず `get-all-tasks` で既存タスク一覧（完了含む）を取得する
2. 会話の内容と既存タスク名を照合する
   - 完全一致でなくてよい。キーワードの重複で判断する
   - 例: 「フロントハブ強度UP組込み完了」→ T0003「フロントホイールのハブの剛性を上げ…」
   - 例: 「コネクタ固定ブラケットは作り直して」→ T0018「コネクタ固定ブラケットの改修…」
3. 該当する既存タスクがあれば `update-progress` で進捗を追記
4. 該当タスクがなければ `add-task` で新規追加

## Google Sheets操作

スプレッドシートIDの取得:
```bash
SPREADSHEET_ID=$(cat /workspace/group/.spreadsheet_id 2>/dev/null)
```

### タスク一覧取得（完了・中止含む全件）
```bash
SPREADSHEET_ID="$SPREADSHEET_ID" node /app/gsheets.js get-all-tasks
```

### 進捗の追記（最重要）
```bash
SPREADSHEET_ID="$SPREADSHEET_ID" node /app/gsheets.js update-progress "タスク名の一部" "YYYY/MM/DD：進捗内容" ["新ステータス"]
```
- 進捗メモは既存の内容に改行で追記される
- ステータスは省略可（省略時は変更なし）
- 日付はDiscordメッセージのタイムスタンプから取得すること

### ステータスのみ更新
```bash
SPREADSHEET_ID="$SPREADSHEET_ID" node /app/gsheets.js update-status "タスク名の一部" "完了"
```

### タスク追加
```bash
SPREADSHEET_ID="$SPREADSHEET_ID" node /app/gsheets.js add-task "タスク名" "カテゴリ" "優先度" "期限" "担当者"
```

### 一括更新（複数の変更がある場合）
```bash
SPREADSHEET_ID="$SPREADSHEET_ID" node /app/gsheets.js batch-update '[
  {"taskHint":"ハブ剛性","progress":"2026/03/07：組込完了","status":"完了"},
  {"taskHint":"カメラ","progress":"2026/01/31：カメラ設置完了"}
]'
```

## 応答ルール

### タスクを検出した場合
1. スプレッドシートを更新する
2. 更新内容のサマリーをDiscordに投稿する（短く、1〜3行程度）
   例: `✅ T0003「フロントホイールのハブ剛性UP」を完了に更新しました`
   例: `📝 新規タスク T0018「コネクタ固定ブラケット改修」を追加しました（メカ/中/honey）`

### タスクを検出しなかった場合
**何も応答しない**。`<internal>タスク検出なし</internal>` のみ出力する。
雑談に反応してはいけない。

### 複数のタスク関連メッセージがある場合
まとめて処理し、変更サマリーをまとめて1回で投稿する。
batch-updateを積極的に使い、API呼び出し回数を減らすこと。
